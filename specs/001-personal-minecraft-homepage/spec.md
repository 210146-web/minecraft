# 規格：個人 Minecraft 首頁（草案）

## 1. 背景
此功能為個人使用（給自己用），以最小可行產品為主。

## 2. 目標
建立一個可部署在 GitHub Pages 的前端靜態網站首頁，用來集中顯示個人 Minecraft 相關資訊。

## 3. 非目標
- 不做後端服務
- 不做登入/權限系統
- 不做資料庫整合

## 4. 使用者故事
- 作為站點擁有者，我希望開啟首頁就能看到整理好的 Minecraft 內容區塊。
- 作為站點擁有者，我希望可直接部署到 GitHub Pages。

## 5. 功能需求（可驗收）
1. 網站必須為純前端靜態資產（HTML/CSS/JS），不依賴後端執行。
2. 首頁必須至少包含：站點標題、簡介、Minecraft 內容區塊（至少 3 個項目）。
3. 版面需支援桌機與手機瀏覽（RWD）。
4. 專案需可直接以 GitHub Pages 方式部署（使用根目錄或 docs 目錄皆可）。

## 6. 驗收方式
1. 在本機直接開啟首頁可看到完整內容與基本樣式。
2. 在手機寬度（約 375px）下版面仍可閱讀且無水平捲軸。
3. 專案推送至 GitHub 後可按既定 Pages 設定成功發佈。

## 7. 假設與待確認
- 目前未提供更細節需求（例如配色、內容來源、互動需求），先以靜態展示為主。
- 若後續補充需求，會在此規格上增量更新，不重寫整份文件。
